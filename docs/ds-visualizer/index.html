1. 背景：Action Space（动作空间）与 SAC

先看随机策略在什么坐标里探索

OGBench 5D 动作里，前三维是末端位置增量 dx, dy, dz，通常是 expressed in world frame 的 EE position delta；后两维是 yaw 和 gripper。SAC 在这个动作空间上学习随机策略。

1
Action Space（动作空间）先决定方向语义 Delta World 的方向是全局坐标方向；Delta EE 的方向是末端局部方向。本文讨论的是前三维位移向量上的方向与速度。
2
Critic（价值网络）拟合 TD Target（时序差分目标） Q(s,aπ) 评估训练用 policy action（策略动作），训练目标是 Bellman backup（贝尔曼备份）形成的一步 TD target；采样的下一步动作来自当前 actor（策略网络）。
3
Actor（策略网络）输出 μ 和 σ μ 表示动作分布中心，偏 exploit（利用）；σ 表示采样尺度，偏 explore（探索）。问题在于 σ 的坐标语义是否可解释。
4
Explore / Exploit（探索 / 利用）要能被解释 如果 σ 只是在 XYZ 坐标轴上变大，就很难说它增加的是方向探索、速度探索，还是二者的混合。

2. 动机：XYZ 表示不等于任务语义

问题不是维度，而是坐标系

直接用 TanhNormal 在 dx,dy,dz 上采样可以运行，但三维 Cartesian 坐标不是任务真正关心的语义坐标。位移动作同时包含方向和速度。

1. 方向和速度纠缠 (dx,dy,dz) 同时决定方向 v/||v|| 和速度 ||v||，一次噪声会同时改变两个语义。
2. σ 先形成动作置信椭球 Cartesian 高斯的 σx,σy,σz 定义的是固定置信水平下的动作采样椭球，而不是直接定义方向探索。
3. μ 的长度会改变方向探索 同样置信水平、同样体积的蓝色椭球，离原点越近，从原点看过去张开的方向视锥越大。
4. chunk 会重复这个问题 H=5 时每一步都有一组三维位移；方向、速度和标量动作的边界更需要清楚。

3. 可视化：固定 σ 如何变成不同方向视锥

Three.js（3D 可视化）交互视锥

蓝色球/椭球表示 Cartesian 高斯在 [dx,dy,dz] 空间里的固定置信水平采样区域，例如 95% 概率质量对应的一层等概率置信椭球；红色锥体表示从原点看过去，这个局部采样区域诱导出的方向探索范围。 保持 σ 不变时，只有 l=||μ|| 变小，方向角也会被放大。

这里的"95% 置信椭球"指联合置信区域（Mahalanobis 距离 ≤ χ²_D(p)，保证总概率质量 = 95%），不是"每维 95% 区间拼成的长方体"——后者只包含 0.95³ ≈ 85.7% 的联合质量。两者形状、含义都不同，不能混用。

视锥参数

θ ≈ asin(σ / l)

这里把均值方向固定在 x 轴，只改变均值长度和探索半径，用来隔离说明方向噪声为何会随动作幅度变化。

l 越小，蓝色置信椭球越接近原点，红色方向视锥越容易张开。
σ 表示 Cartesian 三维高斯的局部采样尺度；在 DS 中，方向 σ 和速度 σ 会被分开解释。
11.5° 方向半角 θ。越大表示同样采样噪声会造成更剧烈的方向变化。
0.20 σ / l。这个比值越高，Cartesian 探索越不像“沿原方向微调”。

这正是 direction-speed 分层的动机：让方向维度的 σ 直接控制方向探索视锥，让速度维度的 σ 只影响速度探索， 不再让 ||μ|| 隐式改变方向探索强度。

5. 概率参数化：Action（动作）层级与方向分布

Integrated（分布内集成） · Adapter（动作适配器）

spherical（球坐标）、stereographic（立体投影）和 tangent（切空间）是 Integrated DS（分布内集成）的三种方向分布；Adapter DS（动作适配器）来自 direction_speed/adapter.py，负责 policy/env action（策略动作 / 环境动作）坐标转换。

5.1 三种 Action（动作）变量

从 Actor（策略网络）采样到 env.step

下面的差别只回答一个问题：SAC 训练看到的 policy action（策略动作）和环境原生 env action（环境动作）是不是同一个变量。 Baseline（基线）中二者相同；Integrated DS（分布内集成）中通常也相同；但 Integrated DS 仍然可以和 Adapter DS（动作适配器）组合，此时还要看后面的执行层 g。

z / ξ latent value（潜在变量） Actor（策略网络）从 Gaussian（高斯分布）采样出的未 squash（压缩）值。Baseline（基线）常写 z，Integrated DS（分布内集成）常写 ξ。
aπ policy action（策略动作） Actor（策略网络）暴露给 SAC loss（损失函数）、log_prob（对数概率）和 Critic（价值网络）的动作。Baseline（基线）中是 tanh(z)，Integrated DS（分布内集成）中是 f(ξ)；Adapter DS（动作适配器）中是 compose（合成）前的 â。
a_env env action（环境动作） 环境原生动作，也就是 dataset（数据集）/ replay buffer（回放缓冲区）/ env.step 真正使用的 5D 动作。
Baseline（基线） aπ = a_env = tanh(z)
Integrated DS（分布内集成） aπ = f(ξ); 仅当 g = identity 时 a_env = aπ
Adapter DS（动作适配器） aπ = DS coordinates, a_env = compose(aπ)
Integrated DS + Adapter DS（分布内集成 + 动作适配器） aπ = f(ξ), a_env = compose(aπ)

Adapter DS（动作适配器）实现

  • decompose: xyz -> unit_direction + speed_coord，其余维度原样保留。
  • speed_coord = 2 * (||xyz|| / R(u)) - 1，其中 R(u)=1/max(|u_i|)。
  • compose: 先 normalize direction，再用 ρ=(speed_coord+1)/2 还原 xyz=ρR(u)u。
D+1 表示 非可逆 FQL 可用

Adapter（动作适配器）核心映射

r = ||xyz||
u = xyz / max(r, eps)
R(u) = 1 / max(abs(u))
speed_coord = 2 * (r / R(u)) - 1

ρ = (speed_coord + 1) / 2
xyz = normalize(u) * R(u) * ρ

a_env = clip([xyz, other], -1, 1)

Q(s,aπ) 和 log π(aπ|s)
内部都用 decomposed action a_hat。

Spherical（球坐标）实现

  • actor base distribution 仍是 D 维 Gaussian，但前三维 latent 解释为 theta_raw, phi_raw, speed_raw。
  • bounded sigmoid 将 theta 映射到 (0,π)，phi 映射到 (0,2π)，speed 映射成沿 cube radial bound R(u) 的归一化比例。
  • log-det 包含球坐标体积项 r² sin(theta) 和三条 sigmoid 的导数。
可逆 bijector Jacobian-corrected 极点附近更敏感

Spherical（球坐标）核心映射

ξ = [theta_raw, phi_raw, speed_raw, ...]
theta = π * sigmoid(theta_raw)
phi = 2π * sigmoid(phi_raw)
direction = [sin(theta)cos(phi), sin(theta)sin(phi), cos(theta)]
R = 1 / max(abs(direction))
r = R * sigmoid(speed_raw)

x = r sin(theta) cos(phi)
y = r sin(theta) sin(phi)
z = r cos(theta)

aπ = bijector.forward(ξ)
a_env = g(aπ)  # Integrated DS: g = identity

Stereographic（立体投影）实现

  • 前三维 latent 是 plane_x, plane_y, speed_raw，先在平面采样，再 inverse stereographic 到单位球。
  • 方向映射没有球坐标角度极点，实验中作为 RLPD/SAC 的 DS 主线。
  • log-det 使用球极投影面积项 4 / (1 + rho²)²，再乘上 r²、R(u) 和 speed sigmoid 导数。
推荐主线 可逆 bijector 方差更稳

Stereographic（立体投影）核心映射

ξ = [px, py, speed_raw, ...]
rho2 = px² + py²
direction = [
  2px / (1 + rho2),
  2py / (1 + rho2),
  (1 - rho2) / (1 + rho2)
]

speed = R(direction) * sigmoid(speed_raw)
action_xyz = speed * direction
aπ = bijector.forward(ξ)
a_env = g(aπ)  # Integrated DS: g = identity

Tangent（切空间）实现

  • actor 学习平均方向 μ(s)，方向噪声不再使用固定的全局角度或平面坐标。
  • 在 μ 的二维切平面采样各向同性 Gaussian，再通过 sphere exponential map 返回单位球面。
  • 角距离只由切向噪声长度决定，与 μ 位于北极、赤道或南极无关。
旋转不变探索 Jacobian-corrected 对跖点 cut locus

Tangent（切空间）核心映射

ε ~ Normal(0, σ² I₂)
q = ||ε||
r = (π-δ) tanh(q / (π-δ))
v = B_μ (r / q) ε

direction = Exp_μ(v)
          = cos(r) μ + sinc(r) v

log|J_dir| = log(sin(r) / q · dr/dq)
speed = R(direction) * sigmoid(speed_raw)
aπ = a_env = [speed * direction, ...]

6. Speed Bound（速度边界）：Sphere（球）与 Cube（立方体）

相同方向 u · 相同速度比例 ρ

速度比例 ρ 独立于方向分布。Sphere 使用固定半径 R(u)=1；Cube 使用 R(u)=1/max|uᵢ|，因此沿斜方向能够覆盖单位球之外、cube 之内的合法动作。

黄色 = sphere action · 蓝色 = cube action 虚线表示同一单位方向；拖动方向可观察 cube 径向边界随 u 改变。
Rcube(u) = 1 / max(|ux|, |uy|, |uz|)

7. Modular Pipeline（模块化流水线）：从 action_raw 到 action_env

先看数据流，再看实现层级

Adapter DS（动作适配器）在 policy/env action（策略动作 / 环境动作）之间转换；Integrated DS（分布内集成）在 Actor（策略网络）概率分布内部改变采样坐标并计算 Jacobian（雅可比）。Integrated DS 可以直接输出 env action，也可以继续接 Adapter（适配器）。

Overview（总览）

总览两个层级：Adapter DS（动作适配器）决定 aπ 如何变成 a_env；Integrated DS（分布内集成）决定 latent ξ（潜在变量）如何变成可计算 log_prob（对数概率）的 aπ。
